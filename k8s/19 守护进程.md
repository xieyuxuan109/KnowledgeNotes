# 一、什么是 DaemonSet？

DaemonSet 是 Kubernetes 控制器的一种。

作用：

在每个 Node 上运行一个 Pod  
新增 Node 自动创建  
删除 Node 自动清理  

适用场景：

- 日志采集（fluentd / filebeat）
- 监控（node-exporter）
- 网络插件（calico）
- 存储插件

---

# 二、创建 DaemonSet

##  使用 YAML 创建

```bash
kubectl apply -f daemonset.yaml
```

示例：

```yaml
apiVersion: apps/v1
kind: DaemonSet
metadata:
  name: node-exporter
spec:
  selector:
    matchLabels:
      app: node-exporter
  template:
    metadata:
      labels:
        app: node-exporter
    spec:
      containers:
      - name: node-exporter
        image: prom/node-exporter
```
---

# 三、查看 DaemonSet

##  查看列表

```bash
kubectl get daemonsets
kubectl get ds
```

输出示例：

```
NAME            DESIRED   CURRENT   READY
node-exporter   3         3         3
```

解释：

- DESIRED = 期望数量（节点数）
- CURRENT = 当前运行
- READY = 正常运行

---

##  查看详情

```bash
kubectl describe ds node-exporter
```

---

##  查看对应 Pod

```bash
kubectl get pods -o wide
```

查看运行在哪些节点：

```bash
kubectl get pods -o wide | grep node-exporter
```

---

# 四、删除 DaemonSet

##  删除 DaemonSet

```bash
kubectl delete ds node-exporter
```

效果：

- 所有 Pod 同时删除

---

##  保留 Pod 删除控制器

```bash
kubectl delete ds node-exporter --cascade=orphan
```

效果：

- 删除 DaemonSet
- 保留已创建的 Pod

---

# 五、更新 DaemonSet

##  修改镜像

```bash
kubectl set image ds/node-exporter node-exporter=prom/node-exporter:v1.7.0
```

---

##  查看滚动更新状态

```bash
kubectl rollout status ds/node-exporter
```

---

##  查看历史版本

```bash
kubectl rollout history ds/node-exporter
```

---

##  回滚

```bash
kubectl rollout undo ds/node-exporter
```

# 六、控制调度

##  只在特定节点运行

打标签：

```bash
kubectl label node node1 disk=ssd
```

DaemonSet 中指定：

```yaml
nodeSelector:
  disk: ssd
```

---

##  查看节点标签

```bash
kubectl get nodes --show-labels
```

---

##  使用污点与容忍

查看污点：

```bash
kubectl describe node node1 | grep Taint
```

在 DaemonSet 中添加：

```yaml
tolerations:
- operator: Exists
```

# 七、排查问题

##  Pod 未运行

```bash
kubectl describe pod <pod-name>
```

---

##  查看日志

```bash
kubectl logs <pod-name>
```

多容器：

```bash
kubectl logs <pod-name> -c <container-name>
```

---

##  查看事件

```bash
kubectl get events --sort-by=.metadata.creationTimestamp
```
