# Kubernetes 核心概念详解

## 一、核心架构与基础

### 1. 集群 (Cluster)
Kubernetes 部署的单位就是一个**集群**。它可以看作是一组协同工作的机器集合，共同运行容器化应用。

### 2. 节点 (Node)
集群中的每台机器（物理机或虚拟机）称为一个**节点**。

#### Master Node（控制平面）
集群的"大脑"，负责管理、调度、监控和协调集群中的所有活动。

| 组件                   | 功能描述                                                         |
| ---------------------- | ---------------------------------------------------------------- |
| **API Server**         | 所有操作的唯一入口，用户通过 `kubectl` 或 API 发送的命令都到这里 |
| **etcd**               | 高可用的键值数据库，存储整个集群的配置数据和状态信息             |
| **Scheduler**          | 负责调度 Pod 在哪个 Node 上运行                                  |
| **Controller Manager** | 运行各种控制器，确保当前状态与期望状态一致                       |

#### Worker Node
实际运行容器化应用的工作节点。

| 组件           | 功能描述                                              |
| -------------- | ----------------------------------------------------- |
| **Kubelet**    | 节点上的"代理"，负责与 Master 通信并管理 Pod 生命周期 |
| **Kube-Proxy** | 负责节点的网络规则，实现 Service 的网络路由和负载均衡 |
| **容器运行时** | 真正运行容器的软件，如 Docker, containerd, CRI-O 等   |
