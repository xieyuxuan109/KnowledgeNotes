# Kubernetes v1.32.0 生产级环境部署手册 (Ubuntu 24.04)

本文档记录了在三台 Ubuntu 24.04 机器上，将预装的 v1.34 降级并成功构建 v1.32.0 集群的全过程。

---

## 1. 节点信息预设

| 主机名 | IP 地址 | 角色 |
| :--- | :--- | :--- |
| **k8s-master1** | 192.168.1.222 | 控制平面 (Control Plane) |
| **k8s-node1** | 192.168.1.223 | 工作节点 (Worker Node) |
| **k8s-node2** | 192.168.1.224 | 工作节点 (Worker Node) |

---

## 一、三台机器通用配置（全部执行）
### 设置主机名
```
hostnamectl set-hostname k8s-master
hostnamectl set-hostname k8s-node1
hostnamectl set-hostname k8s-node2
```
### 配置hosts
```
cat >> /etc/hosts <<EOF
10.17.220.220 k8s-master
10.17.220.221 k8s-node1
10.17.220.222 k8s-node2
EOF
```
### 关闭 swap
```
vim /etc/fstab
找到类似这一行：
/swap.img none swap sw 0 0
把这一行前面加 # 注释掉
swapoff -a
swapon --show
```
### 内核参数
```
cat <<EOF > /etc/modules-load.d/k8s.conf
overlay
br_netfilter
EOF

modprobe overlay
modprobe br_netfilter
```
```
cat <<EOF > /etc/sysctl.d/k8s.conf
net.bridge.bridge-nf-call-iptables = 1
net.bridge.bridge-nf-call-ip6tables = 1
net.ipv4.ip_forward = 1
EOF

sysctl --system
```

## 三、安装 Kubernetes 1.35

```
添加源
apt install -y apt-transport-https ca-certificates curl gpg
```
```
mkdir -p /etc/apt/keyrings
curl -fsSL https://pkgs.k8s.io/core:/stable:/v1.35/deb/Release.key \
| gpg --dearmor -o /etc/apt/keyrings/kubernetes-apt-keyring.gpg
```
```
echo "deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] \
https://pkgs.k8s.io/core:/stable:/v1.35/deb/ /" \
> /etc/apt/sources.list.d/kubernetes.list
```
```
安装
apt update
apt install -y kubelet kubeadm kubectl
apt-mark hold kubelet kubeadm kubectl
# 解开sudo apt-mark unhold <package-name>
```
## 四、初始化 Master（只在 master1）
```
（只在 master1）
kubeadm init \
  --apiserver-advertise-address=10.17.220.220 \
  --pod-network-cidr=192.168.0.0/16 \
  --image-repository registry.aliyuncs.com/google_containers
```
```
如果不小心中断了
kubeadm reset -f
reboot
然后重新init
```

## 五、安装 Calico（无代理）只有master1
```
配置 kubectl
mkdir -p $HOME/.kube
cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
chown $(id -u):$(id -g) $HOME/.kube/config
```
```
下载（如果 master1 能访问 GitHub 就直接）：
curl -O https://raw.githubusercontent.com/projectcalico/calico/v3.27.3/manifests/calico.yaml
如果 GitHub 访问慢，可以在有网机器下载再传。
```
```
ctr -n k8s.io images pull docker.m.daocloud.io/calico/cni:v3.27.3
ctr -n k8s.io images pull docker.m.daocloud.io/calico/node:v3.27.3
ctr -n k8s.io images pull docker.m.daocloud.io/calico/kube-controllers:v3.27.3
ctr -n k8s.io images pull docker.m.daocloud.io/calico/pod2daemon-flexvol:v3.27.3
```
```
sed -i 's#docker.io/calico/#m.daocloud.io/docker.io/calico/#g' calico.yaml
sed -i 's#quay.io/calico/#m.daocloud.io/docker.io/calico/#g' calico.yaml
```
```
grep image: calico.yaml
```
```
kubectl apply -f calico.yaml
```
```
kubectl get pods -n kube-system
```
