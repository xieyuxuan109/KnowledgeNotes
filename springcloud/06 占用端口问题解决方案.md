# 占用端口问题解决方案
## 端口占用查询
## 在开发或调试过程中，端口被占用是常见问题。以下是查询端口占用情况并释放端口的步骤。

### 查看所有运行的端口
```
打开命令提示符（以管理员身份运行），输入以下命令查看所有端口使用情况：
>netstat -ano|findstr "9000"
  TCP    0.0.0.0:9000           0.0.0.0:0              LISTENING       18208
  TCP    [::]:9000              [::]:0                 LISTENING       18208
此命令会列出所有活动连接及其对应的PID（进程ID）。找到目标端口，例如 9000
```
### 查询 PID 对应的进程
```
通过以下命令查看具体是哪个进程占用了该端口：

>tasklist |findstr "18208"
java.exe                     18208 Console                    8    120,072 K
结果会显示进程名称，例如 java.exe。
```
### 结束占用端口的进程
```
使用以下命令强制结束对应进程：
taskkill /T /F /PID 18208
/T：终止指定进程及其子进程。
/F：强制终止。
或者，打开任务管理器，切换到“进程”选项卡，通过PID找到对应进程并手动结束。
注意事项
确保以管理员权限运行命令提示符，否则可能无法结束某些系统级进程。
在结束进程前，确认该进程是否为关键任务，以免影响系统稳定性。
通过以上步骤即可快速定位和解决端口占用问题。
```
